## Spacing Scale

The project follows an 8-point scale mapped to Tailwind utility classes. All new Phase 13 elements use these exact values:

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | p-1 / gap-1 | 4px | Icon-to-label gap inside stat chips |
| sm | p-2 / gap-2 | 8px | Gap between chips inside the stats card |
| md | p-3 / gap-3 | 12px | Signal row padding — codebase exception: existing bullish/bearish signal rows use `p-3` at ResearchReport.tsx lines 243, 262. Phase 13 does not introduce new `p-3` uses. |
| md+ | p-4 / gap-4 | 16px | Card internal padding (compact variant) |
| xl | p-6 / gap-6 | 24px | Section-level card padding (matches existing Market Sentiment card) |
| 2xl | space-y-8 | 32px | Between top-level sections in `<main>` (existing) |

Exceptions:
- Sentiment Intelligence card uses `p-4` (compact, 16px) — intentionally tighter than standard `p-6` cards, per D-18 "compact/scannable" directive
- Forward Outlook section uses `p-6` (24px) to match Executive Summary weight
- Stat chip pill padding: `px-4 py-1` (16px / 4px) — matches existing Market Sentiment badge pattern (`px-4 py-1`)

## Typography

All roles use Inter. Monospace data values use JetBrains Mono (`font-mono`). Exactly 4 size levels declared, 2 weights used, matching the established report component pattern.

| Role | Size | Weight | Line Height | Tailwind | Usage in Phase 13 |
|------|------|--------|-------------|----------|-------------------|
| Label / Header | 10px | 700 (bold) | 1.2 | `text-[10px] font-bold tracking-widest uppercase` | Card section headers ("Sentiment Intelligence", "Forward Outlook"), stat chip labels (BULL %, BEAR %, P/C) |
| Body / Annotation | 11px | 400 (regular) | 1.45 | `text-[11px] text-on-surface-variant leading-tight` | Source count annotation ("N community sources"), sub-label under stat chips, confidence-style explanation text |
| Body Standard | 12px (xs) | 400 (regular) | 1.6 | `text-xs text-on-surface-variant leading-relaxed` | Forward Outlook narrative prose (same as Key Risks / Investment Thesis body) |
| Data Value | 14px (sm) | 700 (bold) | 1.0 | `text-sm font-mono font-bold` | Bull %, bear %, put/call ratio numeric value inside stat chips |

Typography constraints:
- No new font sizes beyond these 4. Phase 13 adds no display-level text.
- Tracking: `tracking-widest` on all 10px labels (matches existing `text-[10px] font-bold tracking-widest uppercase` throughout ResearchReport.tsx).
- Monospace reserved for: numeric data values (bull_pct, bear_pct, put_call_ratio), source counts. NOT used for narrative prose.

Accent reserved for:
- `text-secondary` (teal): bull% numeric value and its chip background only
- `text-error` (red): bear% numeric value and its chip background only

- `text-primary` (blue): Forward Outlook section label, left border, and icon only

**Color rule:** Do not apply primary blue to any element in the Sentiment Intelligence card. Primary blue is reserved for the Forward Outlook section to signal it as the capstone synthesis (matching Executive Summary's use of primary blue at the top of the report — bookend treatment).

## Component Inventory

Two new UI elements are introduced in Phase 13. Both are added to `ResearchReport.tsx` only. No new files.

### 1. Sentiment Intelligence Card (D-18)

**Position:** After the Market Sentiment card, before the Bullish/Bearish factors grid — within the left column (`lg:col-span-8`).

**Structure:**
```
[SENTIMENT INTELLIGENCE header]         [optional TRENDING badge]
──────────────────────────────────────────────────────
[ BULL %  chip ] [ BEAR %  chip ] [ P/C RATIO chip ]
──────────────────────────────────────────────────────
  N community sources scraped
```

**Chip anatomy (each of 3 chips):**
- Container: `bg-surface-container-highest px-4 py-2 rounded flex flex-col items-center gap-1`
- Label: `text-[10px] font-bold tracking-widest uppercase text-on-surface-variant` — e.g. "BULL", "BEAR", "P/C RATIO"
- Value: `text-sm font-mono font-bold` + color token per above
  - Bull chip value: `text-secondary`
  - Bear chip value: `text-error`
  - Put/call chip value: `text-tertiary`
- Sub-label (put/call only): `text-[10px] text-tertiary` showing interpretation label — "BULLISH" / "BEARISH" / "NEUTRAL"

**Card container:** `bg-surface-container p-4 rounded-lg`

**Section header:** `text-[10px] font-bold tracking-widest uppercase text-on-surface-variant flex items-center gap-2` with Material Symbol `monitoring` icon in `text-tertiary text-sm`

**Annotation row:** `text-[11px] text-on-surface-variant` — "{N} community sources scraped" — separated from chips by `border-t border-surface-container-highest pt-2 mt-2`

**Null / unavailable handling:**
- If `stocktwits_bull_pct` is null: chip shows "—" in `text-on-surface-variant` instead of a value. Label remains. No chip is hidden — all 3 chips always render.
- If `put_call_ratio` is null: chip shows "—" with no interpretation sub-label.
- If community source count is 0: annotation reads "Community sources unavailable"

### 2. Forward Outlook Section (D-19)

**Position:** Final section in `<main>`, after Sources — the last element before `<FooterTicker />`.

**Structure:** Full-width section (not inside the 12-column grid). Matches Executive Summary card pattern.

**Container:** `bg-surface-container p-6 rounded-lg border-l-4 border-primary relative overflow-hidden`

**Glow accent:** `<div className="absolute top-0 right-0 w-48 h-48 bg-primary/5 blur-[100px]" />` — identical to Executive Summary glow

**Header row:** Material Symbol `auto_awesome` in `text-primary text-base` + label `text-[11px] font-bold tracking-widest uppercase text-primary` — "Forward Outlook"

**Body:** `text-sm text-on-surface leading-relaxed max-w-4xl` — renders `future_projection` string via `<Md>` component for bold support

**Conditional render:** Wrapped in `{future_projection && (...)}` — not rendered if field is absent or empty string (backward compat with stored reports)

## Interaction Contract

Phase 13 adds read-only display elements. No new interactive states beyond what already exists.

| Element | Interaction | State |
|---------|-------------|-------|
| Sentiment Intelligence card | None — read-only display | Static |
| Stat chips (bull/bear/P/C) | None — no hover, no click | Static |
| Forward Outlook section | None — read-only display | Static |
| Null chip values | Render "—" — no tooltip, no click | Static |

No new loading states. The Sentiment Intelligence data is gathered during the existing analysis pipeline — the ResearchProgress component (existing) handles loading. By the time ResearchReport renders, all data is present.
