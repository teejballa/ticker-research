# Phase 30.1 Plan 05: Firecrawl Removal Summary

**Firecrawl removed entirely from src, tests, scripts, and package.json. Permanent CI guard prevents reintroduction. Operator-driven shadow-soak/calibration/UAT artifacts scaffolded but not executed in this session — production already runs on Reddit/Twitter via Xpoz Pro per the 30.1-pivot commits.**

## What landed

### Code-side removal (this session)
- `npm uninstall @mendable/firecrawl-js` — 12 packages removed, lockfile clean
- `COMMUNITY_SCAN_SOURCE` flag and `CommunityScanSource` type deleted from `features.ts`
- `'firecrawl'` removed from every type union: `FieldOrigin`, `ProviderId`, `CipherSource`, `SentimentObservationSource`

- Two test files deleted that no longer apply: `lightweight-community-scan.breaker.integration.test.ts` (Firecrawl-specific breaker integration) + `features-community-scan-source.unit.test.ts` (flag-specific tests)
- Test mocks for `@mendable/firecrawl-js` removed from `analysis-web-mode.test.ts` and `cron-module-load.unit.test.ts`
- `FIRECRAWL_API_KEY` removed from `.env.example` along with the `FEATURE_COMMUNITY_SCAN_SOURCE` variable

### Permanent CI guard (this session)
- `scripts/check-firecrawl-removed.ts` greps `src/`, `tests/`, `scripts/` for `firecrawl|FIRECRAWL|@mendable/firecrawl-js` (case-sensitive, file allowlist for self + alert resolver + calibrate-engagement-tiers + their tests)
- `tests/scripts/check-firecrawl-removed.unit.test.ts` invokes the guard via `tsx` and asserts exit 0
- Wired as `npm run check-firecrawl-removed` in package.json scripts
- **Smoke run:** `[check-firecrawl-removed] PASS — no Firecrawl references in src, tests, scripts/`

### Operator-script scaffolding (this session)
- `scripts/migrations/30.1-resolve-firecrawl-alert.ts` — idempotent `UPDATE ProviderHealthAlert SET resolved_at=NOW() WHERE provider_id='firecrawl' AND resolved_at IS NULL`. Tested with mocked Prisma asserting idempotency + single-call invocation + race-condition exit-1 path.
