# Phase 30.1 Plan 02: Reddit OAuth + Rate Bucket + Per-Sub Search Summary

One-liner: Reddit OAuth client_credentials adapter with Upstash-cached bearer (SETNX-guarded), atomic minute-precision rate bucket (env-driven ceilings, 60/40 default), and Promise.allSettled per-sub search wrapped in the canonical withTelemetry → withBreaker → withRetry composition — ready for Plan 30.1-03 orchestrator wiring.

## What Landed

### Reddit adapter implementation (`src/lib/data/adapters/reddit.ts` — 330 LOC)

**Task 1 — OAuth token mint + Upstash cache + SETNX dogpile guard**

- `mintFreshToken()` — POST `https://www.reddit.com/api/v1/access_token`:
  - `Authorization: Basic ${base64(id:secret)}`
  - `User-Agent: Cipher/1.0 (by /u/cipher-research)` (D-06, MANDATORY)
  - `Content-Type: application/x-www-form-urlencoded`
  - Body: `grant_type=client_credentials`
  - 2-second `AbortController` timeout (T-30.1-02-04 / RESEARCH Pitfall 2)
  - Fail-fast on `REDDIT_CLIENT_ID/SECRET unset`
  - Throws Error with `.status = res.status` on `!res.ok` so withRetry's classifier correctly distinguishes 4xx (surface fast) from 5xx (retry)
- `getRedditToken()` — Upstash cache flow:
  1. `r.get('reddit:oauth_token')` — fast path
  2. On miss, `r.set('reddit:oauth_token:refresh_lock', '1', { ex: 5, nx: true })` — SETNX dogpile guard
  3. If lock NOT acquired: `setTimeout(200ms)` then re-read; if still miss, fall through to mint (orphaned-lock safe)
  4. `withRetry(mintFreshToken)` — 3 attempts, 5xx/network only
  5. `r.set('reddit:oauth_token', token, { ex: max(60, expires_in - 60) })`
  6. `r.del(refresh_lock)` in finally block
  - Graceful degrade: when `getRedis()` returns null (Upstash unconfigured), mint a fresh token every call (no SETNX, no cache write)

**Task 2 — Atomic Upstash rate bucket (env-driven QPM ceilings)**

- `rateCeilingReport()` / `rateCeilingCron()` — function-per-call env reads:
  - `REDDIT_QPM_CEILING_REPORT` (default 60 — full ceiling for synchronous user-facing report-gen)
  - `REDDIT_QPM_CEILING_CRON` (default 40 — leaves ≥20 headroom for report-gen contention; bumpable to 70 after 7d clean per RESEARCH Open Question 1)
- `consumeRateToken(priority)` — atomic minute-precision bucket:
  - Key: `reddit:rate_bucket:${Math.floor(Date.now() / 60000)}`
  - `r.incr(key)` then `r.expire(key, 65)` ONLY when count === 1 (subsequent INCRs skip EXPIRE)
  - Returns `'ok'` if count ≤ ceiling, `'throttle'` otherwise
  - Graceful degrade: `'ok'` when Upstash is null OR INCR rejects (transient — Reddit 429 + breaker would catch real overload independently)
- **Known minute-boundary race** (T-30.1-02-08): accepted — 60 QPM headroom absorbs micro-double-throughput at minute roll-over. Sliding-window upgrade deferred unless production 429s exceed 1%.

**Task 3 — Per-sub search + Promise.allSettled fan-out + composition wrapping**

- `searchOneSubreddit(ticker, sub, token)` (internal):
  - `GET ${OAUTH_BASE}/r/${sub}/search.json?q=${encodeURIComponent(ticker.toUpperCase())}&restrict_sr=on&sort=new&t=week&limit=25`
  - Headers: `Authorization: Bearer ${token}`, `User-Agent: USER_AGENT`, `Accept: application/json`
  - Maps `body.data.children[].data` to `RedditPost[]` with defensive type coercion on all 10 fields
  - Filters out rows with `id === ''` or `created_utc === 0` (malformed/deleted)
  - `!res.ok` → `statusError(prefix, res.status)` with `.status` set (per-sub 4xx soft-skipped via Promise.allSettled)
- `fetchRedditCommunity(ticker, subs, priority='cron')` (exported):
  1. Empty-subs early return → `[]`
  2. `consumeRateToken(priority)` → on `'throttle'`, return `[]` (no token mint, no fetches)
  3. `getRedditToken()` once; on failure, soft-fail to `[]` (crons-never-500 invariant)
  4. `Promise.allSettled` over `subs.map(sub => withTelemetry('reddit', () => withBreaker('reddit', () => withRetry(() => searchOneSubreddit(ticker, sub, token))), { ticker }))`
  5. `flatMap` fulfilled results, drop rejected (per-sub 403/404/breaker-trip soft-skipped)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug in plan acceptance grep] Function-per-call ceiling reads vs const-at-load**

- **Found during:** Task 2 implementation.
- **Issue:** Plan acceptance criteria specified `grep "RATE_CEILING_REPORT = Number(process.env.REDDIT_QPM_CEILING_REPORT" src/lib/data/adapters/reddit.ts` returns one match — implying a module-level const initialized at load time. But Task 2's specified unit test `consumeRateToken honors env overrides` uses `vi.stubEnv('REDDIT_QPM_CEILING_REPORT', '100')` *between* tests and expects the new value to take effect. A const initialized at module load would NOT honor live env stubs — Vitest caches modules unless `vi.resetModules()` is called explicitly. The plan's literal grep contradicts its own test specification.
- **Fix:** Implemented as `function rateCeilingReport(): number { return Number(process.env.REDDIT_QPM_CEILING_REPORT ?? 60); }` — function-per-call read so live env edits (test stubs OR production env flips) take effect on next invocation. The plan's intent (env-driven ceilings) is preserved; only the literal `const RATE_CEILING_REPORT = ...` pattern is replaced with the function form.
- **Files modified:** `src/lib/data/adapters/reddit.ts` lines 165-174.
- **Commit:** a4d55fd (impl); the override test passes verifying the fix.

**2. [Rule 3 - Blocking issue] Live integration test directory excluded from default vitest config**

- **Found during:** Task 3 verification.

The plan's literal verify command `npx vitest run tests/integration/reddit-live.integration.test.ts` doesn't find the test.

The intent of the plan's command is preserved — the file is discoverable, the suite skips correctly when `RUN_LIVE_REDDIT` is unset, and the operator can opt-in with the documented env-gated command.
