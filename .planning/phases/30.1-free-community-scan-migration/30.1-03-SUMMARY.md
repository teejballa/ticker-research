# Phase 30.1 Plan 03: Wave 2 — HN Adapter + Flag-Gated Orchestrator Summary

## One-liner

HackerNews Algolia adapter + community-scan orchestrator rewritten with `COMMUNITY_SCAN_SOURCE`-gated branches; legacy Firecrawl path preserved byte-equivalent, new Reddit+HN path produces `EnrichedSnapshot` with optional `reddit_posts` / `hackernews_stories`, shadow mode runs new path via `next/server`'s `after()` for telemetry observation while returning the canonical Firecrawl shape.

## What shipped

### Task 1 — HackerNews Algolia adapter (`src/lib/data/adapters/hackernews.ts`)

- `fetchHackerNewsStories(ticker): Promise<HNStory[]>` calls `https://hn.algolia.com/api/v1/search?query={ticker}&tags=story&numericFilters=created_at_i>{weekAgo}&hitsPerPage=25`.
- Composition stack identical to `apewisdom.ts`: `cached() → withTelemetry('hackernews') → withBreaker('hackernews') → withRetry()` with 3 attempts.
- 10-min cache via `${CACHE_KEYS.community(ticker)}:hackernews` namespace.
- Defensive coercion + filter: hits missing `objectID` or `created_at_i` dropped; `permalink` synthesized as `https://news.ycombinator.com/item?id={objectID}`.
- 5xx retried 3x, 4xx fast-failed (incl. 429), all failures degrade to `[]` (NEVER throws — Phase 19-C-05 T-19-C-05-01 pattern).

### Task 2 — Engagement-tier classifier (`src/lib/data/lightweight-community-scan.ts` + `HYPERPARAMETERS.md`)

- `toEngagementFromFields({ score, num_comments }): 'high' | 'medium' | 'low'` replaces the Firecrawl-era regex-extracted `rawEngagementCount` + `toEngagement` approximation.
- Exported `ENGAGEMENT_TIER_THRESHOLDS` constants (high_score=100, high_comments=50, medium_score=20, medium_comments=10) so plan 30.1-05 calibration is a one-location edit.
- HYPERPARAMETERS.md gains §Community-Engagement Tiers with threshold table, calibration procedure, and `**Last calibration:** TBD — finalize after shadow soak (plan 30.1-05).` placeholder.

### Task 3 — Flag-gated orchestrator (`src/lib/data/lightweight-community-scan.ts`)

- `lightweightCommunityScan(ticker, priority?='cron')` reads `COMMUNITY_SCAN_SOURCE` once and dispatches:
  - `'firecrawl'` → `runFirecrawlPath` (legacy 5-sub Firecrawl + StockTwits + Quiver fan-out, verbatim extraction from pre-plan body).
  - `'reddit'` → `runRedditPath` (parallel `Reddit + HN + StockTwits + StockTwitsRaw + Quiver + Yahoo marketCap`; each adapter has a `.catch()` so one failure does not block siblings — Crons-never-500 invariant).
  - `'shadow'` → returns Firecrawl synchronously; fires `runRedditPath` in the background via `next/server`'s `after()` (Vercel-honored lambda lifetime extender). `setImmediate` was rejected because Vercel may exit the function before the immediate callback fires.
- `EnrichedSnapshot` extended additively (D-20): two new OPTIONAL fields, `reddit_posts?: RedditPost[]` and `hackernews_stories?: HNStory[]`, populated by the Reddit branch and `undefined` on the Firecrawl branch. Plan 30.1-04's `SentimentObservation` writer reads them via optional chaining.
- Highlight assembly (D-19, D-24): one highlight per subreddit that produced ≥1 post; one HN highlight if HN returned ≥1 story. Each highlight gets `standout_quote` = top-scoring post/story title sliced to 140 chars, and `standout_url` = `https://www.reddit.com${permalink}` or `https://news.ycombinator.com/item?id=${objectID}`.
- Subs argument to `fetchRedditCommunity`: `[...COMMUNITY_SUBS.map(s => s.name), upper]` — 17 entries (16 fixed + 1 ticker-niche).
- `REDDIT_CLIENT_ID/SECRET` pre-flight: returns `null` + console.warn when source=reddit but creds unset (mirrors the legacy `if (!process.env.FIRECRAWL_API_KEY) return null;` short-circuit).
- `CommunityHighlight` (in `src/lib/types.ts`) gains OPTIONAL `standout_url?: string` field (D-24 prep — plan 30.1-04 wires it into the report renderer).

## Crons-never-500 invariant

`runRedditPath` wraps every adapter call in `.catch()` (Reddit, Yahoo, Quiver) or trusts adapters with array-valued null-safety contracts (HN, StockTwits). `Promise.all` is used — but every `.catch()` converts to a safe sentinel before the await resolves, so a single upstream failure cannot reject the outer Promise. Same shape as plan 30.1-02's reddit adapter `Promise.allSettled` pattern; same invariant outcome.

## Operator playbook — shadow soak (plan 30.1-05 will execute)

1. **Set the flag in Vercel prod env:** `FEATURE_COMMUNITY_SCAN_SOURCE=shadow`. Redeploy.
2. **Verify telemetry is firing for both paths:** in `/insights/sentiment-health`, the ProviderCallLog dashboard should show new `provider_id='reddit'` and `provider_id='hackernews'` rows alongside the existing `firecrawl` rows. If Reddit/HN rows are absent, confirm `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET` are set in env.
3. **Run shadow for ≥7 days** to accumulate enough watchlist sweeps to compute tier distribution.
4. **Compute tier distributions** — separate dashboards for Firecrawl-era highlights vs Reddit/HN highlights. Each tier (`high` / `medium` / `low`) should land within ±10pp of the Firecrawl-era distribution.
5. **If distribution drifts outside ±10pp**, adjust `ENGAGEMENT_TIER_THRESHOLDS` in `src/lib/data/lightweight-community-scan.ts` and update the HYPERPARAMETERS.md table accordingly. Update `**Last calibration:**` line with date.
6. **Flip to full cutover:** `FEATURE_COMMUNITY_SCAN_SOURCE=reddit`. Redeploy. Watch alpha-vs-SPY drift in `/insights` to confirm no regression in research quality vs Firecrawl-era priors.
7. **Plan 30.1-05** removes Firecrawl entirely from the orchestrator and `package.json`.
