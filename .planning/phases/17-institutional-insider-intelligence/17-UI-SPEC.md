# Phase 17 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds two UI surfaces: (1) the **Smart Money Intelligence** report section
> (new top-level section in ResearchReport between Community Intelligence and Engine
> Calibration), and (2) the **QuadClassPanel** extension to EngineCalibrationPanel
> (4-column grid replacing the 2-column DualClassPanel) plus a 4-class HorizonTable.
> These surfaces extend existing components — the design contract is additive and
> inherits every token and pattern from the established design system.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline badge padding (`px-1 py-0.5`) |
| sm | 8px | Compact element spacing, MetricCard internal row gaps (`gap-1.5`, `gap-2`) |
| md | 16px | Section padding (`p-4`), column divider gap (`gap-4`) |
| lg | 24px | Panel outer padding (`p-6`), column grid gap (`gap-6`) |
| xl | 32px | Between major sections in ResearchReport (`space-y-8` = 32px) |
| 2xl | 48px | (reserved for page-level breaks, not used in these surfaces) |
| 3xl | 64px | (not used in these surfaces) |

Exceptions: 44px touch targets on agreement badge (min-height via `py-2 px-3` at mobile scale). Quarter-unit 2px used for border widths only — not a spacing unit.

Source: derived from existing component patterns in `EngineCalibrationPanel.tsx` and `ResearchReport.tsx`. Confirmed as 8-point scale.

## Typography

All sizes are px equivalents of Tailwind v4 font-size utilities (1rem = 16px at browser default).

| Role | Size | Weight | Line Height | Class |
|------|------|--------|-------------|-------|
| Eyebrow / section label | 10px | 700 (bold) | 1.4 | `text-[10px] font-bold tracking-widest uppercase` |

- `--color-error` (red): OPPOSED agreement badge, DEPRECATED status, empty-state placeholder border when data is completely absent from both classes

Source: established token palette from `globals.css` and existing usage in `EngineCalibrationPanel.tsx`. Extended with institutional/insider color identity assignment (new in this phase).

## Component Inventory

### A. QuadClassPanel (extends DualClassPanel)

**Location:** `src/components/EngineCalibrationPanel.tsx`

**Layout:** Responsive CSS grid.
- Mobile (default): 1 column, cards stacked vertically
- `md:` (768px+): 2 columns, DIFFUSION + TECHNICAL top row, INSTITUTIONAL + INSIDER bottom row
- `lg:` (1024px+): 4 columns, all classes side-by-side with 1px vertical dividers between each

**Grid class:** `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-6`

**Column identity:**
1. DIFFUSION — existing, unchanged
2. TECHNICAL — existing, unchanged
3. INSTITUTIONAL — new, `--color-secondary` identity
4. INSIDER — new, `--color-tertiary` identity

**AgreementBadge placement:** Centered above the 4-column grid in a flex row with eyebrow labels at each end. At lg+: a single row with `DIFFUSION | [badge] | INSTITUTIONAL`. At md: two rows, one per pair. The badge generalizes to N-way logic (see Copywriting Contract).

**Vertical dividers:** `w-px self-stretch bg-outline-variant/30` — one between each adjacent column at lg+, hidden at mobile/md.

**Graceful degradation:**
- Old persisted reports with no `institutional_at_report` / `insider_at_report`: render `DiffusionOnlyPanel` (existing path, unchanged).
- Reports with one class populated and the other null: render populated column normally; null column shows `—` in all metric cells and `NO DATA` status pill.
- Reports with both new classes null but `horizon_calibrations` present: render `DualClassPanel` (Phase 16 path).

**Pattern label maps (new):**

### C. SmartMoneyIntelligence section (new report section)

**Location:** `src/components/ResearchReport.tsx`

**Position:** Between Community Intelligence section and Engine Calibration section.

**Outer container:**
```
section
  className="bg-surface-container border border-surface-container-high p-6 rounded-lg"
  data-testid="smart-money-intelligence"
```

**Section header row** (matches existing section header pattern):
```
flex items-center justify-between mb-5
  left: icon (material-symbols-outlined "account_balance", color text-secondary) + h3 label
  right: filing age indicator (see Copywriting Contract)
```

Header label: `"Smart Money Intelligence"` in `text-[11px] font-bold tracking-widest uppercase text-secondary`

**Two sub-cards layout:** `grid grid-cols-1 md:grid-cols-2 gap-4` inside the section body.

**Sub-card structure (each):**
```
div
  className="bg-surface-container-high p-4 rounded-lg border border-surface-container-highest"
  data-testid="[institutional-flow-card | insider-activity-card]"
```

Each sub-card contains:
1. **Sub-card header:** icon + eyebrow label + filing age chip (right-aligned)
2. **Bucket pill:** pattern bucket name in `font-mono text-xs font-bold` + status badge matching `STATUS_BADGE` map
3. **Activity list:** up to 3 recent filings shown as compact rows (see item format below)
4. **Engine bucket reference:** `"Engine bucket: [BUCKET_LABEL]"` in `text-[10px] text-on-surface-variant tracking-widest uppercase`

**Sub-card: Institutional Flow**
- Icon: `"account_balance"` in `text-secondary`
- Eyebrow: `"INSTITUTIONAL FLOW"` in `text-[10px] font-bold tracking-widest uppercase text-secondary`
- Activity row format: `[fund name or "N institutions"] [action verb] [share delta] · [filing date]`
- Filing age chip: `"Latest 13F: {N} days ago"` — rendered as `text-[10px] font-mono text-on-surface-variant` inline pill

**Sub-card: Insider Activity**
- Icon: `"person_search"` in `text-tertiary`
- Eyebrow: `"INSIDER ACTIVITY"` in `text-[10px] font-bold tracking-widest uppercase text-tertiary`
- Activity row format: `[insider name] ([title]) [action verb] [shares] · [filing date]`
- Filing age chip: `"Latest Form 4: {N} days ago"` — same style

**Null sub-card (AC4 — one class missing):**
```

  className="bg-surface-container-high p-4 rounded-lg border border-error/10"
  data-testid="[institutional-flow-placeholder | insider-activity-placeholder]"
```
Contains only the sub-card header (icon + eyebrow) and the empty-state copy (see Copywriting Contract). No bucket pill, no activity list.

**Both-null state (AC4 edge):** Section renders with both sub-cards showing their respective null placeholders — section header still visible so user knows the feature exists for this ticker.

## Interaction Contract

### QuadClassPanel responsive behavior
- At `lg` (1024px+): 4-col grid visible. All posterior columns visible. CI columns visible.
- At `md` (768–1023px): 2-col grid. CI columns hidden (`hidden xl:table-cell` in HorizonTable).
- Below `md`: single column stack. CI columns hidden.
- Old report degradation: no `horizon_calibrations` → `DiffusionOnlyPanel` (existing path, no change).
- One-class degradation: populated column renders normally; null column renders `—` cells + `NO DATA` pill.

### SmartMoneyIntelligence asymmetric rendering (AC4)
- Insider data present, institutional null → Insider sub-card renders fully; Institutional shows null placeholder.
- Institutional data present, insider null → Institutional sub-card renders fully; Insider shows null placeholder.
- Both null → both sub-cards show null placeholders. Section header still visible.
- Both present → both sub-cards render fully.

### Filing age visual emphasis
- `data_age_days` < 30: render chip in `text-on-surface-variant` (normal).
- `data_age_days` 30–59: render chip in `text-tertiary` (amber — staleness caution).
- `data_age_days` >= 60: render chip in `text-error` (red — stale, 13F is one full quarter old).

### Hover / focus
- No new hover states introduced beyond existing `bg-surface-container` → `bg-surface-container-high` convention.
- No new modal, drawer, or toast interactions.
