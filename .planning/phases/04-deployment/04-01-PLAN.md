    - setup.sh exits 1 with message containing "ANTHROPIC_API_KEY" when that var is not set
    - setup.sh exits 1 with message containing "Python 3.10+" when no qualifying Python binary is found
    - setup.sh exits 1 with message containing "Node.js 18+" when node is missing
    - preflight.test.ts validates these by invoking setup.sh with controlled env vars via child_process.spawnSync
  </behavior>
  <action>
Create `scripts/setup.sh` using the template from RESEARCH.md §Code Examples (setup.sh section). The script must:
1. Check `node --version` — if absent or below 18, print `ERROR: Node.js 18+ required. Install from https://nodejs.org` and exit 1.
2. Check `python3` and `python` candidates with regex `Python 3\.(1[0-9]|[2-9][0-9])` — if none found, print `ERROR: Python 3.10+ not found. Install from https://www.python.org` and exit 1.
3. Check `${ANTHROPIC_API_KEY:-}` is non-empty — if missing, print multi-line error with `.env.local` instructions and exit 1.
4. On success, print `All prerequisites met.` and exit 0.
5. Add shebang `#!/usr/bin/env bash` and `set -euo pipefail`.
6. Make executable: `chmod +x scripts/setup.sh`.

Create `src/lib/__tests__/preflight.test.ts` using Vitest and `child_process.spawnSync`. Each test calls `spawnSync('bash', ['scripts/setup.sh'], { env: {...} })` with a controlled environment:
- Test: all valid env — `status` is 0, stdout contains "All prerequisites met"
- Test: missing ANTHROPIC_API_KEY — `status` is 1, stdout/stderr contains "ANTHROPIC_API_KEY"
- Test: Python unavailable (pass `PATH` without python) — `status` is 1, output contains "Python 3.10+"
- Note: Node check cannot be easily unit-tested (current process IS Node); skip Node version test and note it.
Use `spawnSync` (sync, not async) for simplicity. Set `encoding: 'utf8'` on spawn options. Pass `cwd: process.cwd()` to resolve the script path correctly.
