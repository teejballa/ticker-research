if (process.env.DEPLOYMENT_MODE === 'cloud') {
  const containerUrl = process.env.DAYTONA_CONTAINER_URL;
  if (!containerUrl) {

**Step 2 — Implement the cloud branch (GREEN).**

Modify `src/app/api/analysis/[ticker]/route.ts`:

1. Add `export const dynamic = 'force-dynamic';` at the top, before `maxDuration`. This is CRITICAL (RESEARCH.md §Pitfall 2) — without it, Vercel may cache the env var at build time and the cloud branch never activates in production.

2. Change `maxDuration` from 600 to 300 (Vercel Hobby limit — aligns with `vercel.json`). The container handles the actual long-running work; the Vercel route is just a proxy.

3. At the top of the `POST` handler body (before the `ReadableStream` constructor), add the cloud branch from RESEARCH.md §Pattern 5:
   - Read `process.env.DEPLOYMENT_MODE`
   - If `=== 'cloud'`: read `process.env.DAYTONA_CONTAINER_URL`
     - If missing: return `new Response(JSON.stringify({ type: 'error', message: 'DAYTONA_CONTAINER_URL is not configured.' }), { status: 500, headers: { 'Content-Type': 'application/json' } })`
     - If present: read `filePath` from request body, call `fetch(containerUrl + '/api/analysis/' + ticker, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ filePath }) })`, return `new Response(upstream.body, { headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' } })`
   - If not `'cloud'`: fall through to existing `ReadableStream` / `spawn` logic (unchanged)

The `ticker` value for the proxy URL: extract it from `await params` at the top of the handler (already present in the existing code as `await params` — destructure it: `const { ticker } = await params`).

Run `npm test -- src/app/api/analysis/__tests__/route.test.ts` — all tests including new cloud tests must pass.

**Note on src/app/api/research/[ticker]/route.ts:** Add `export const dynamic = 'force-dynamic';` to this route as well (it reads env vars at runtime). The research route does NOT need a cloud proxy branch — in cloud mode, only the analysis route proxies to the container. The research route (data collection via yahoo-finance2 + Anthropic) runs on Vercel directly. Add the `dynamic` export and run full test suite.
