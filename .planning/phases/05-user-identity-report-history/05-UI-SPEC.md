## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (e.g., sentiment chip inner padding) |
| sm | 8px | Compact element spacing — row internal padding, gap between action buttons |
| md | 16px | Default element spacing — row horizontal padding, section internal gaps |
| lg | 24px | Section padding — RESEARCH HISTORY section top margin |
| xl | 32px | Layout gaps — between home page sections (search → history) |
| 2xl | 48px | Major section breaks — bottom padding below history list |
| 3xl | 64px | Page-level spacing — reserved for scroll scene sections (not this phase) |

Exceptions:
- Nav bar height stays at 44px (h-11 = 44px) matching existing fixed nav
- History row height: 40px tall to allow tap/click target on dense list (not standard 8pt grid but matches terminal density)
- Email truncation container: max-width 192px (24ch × 8px per char average)

## Component Inventory

### New Components

#### `<NavIdentity />`
- Location: inline in `src/app/page.tsx` nav (or extracted as `src/components/NavIdentity.tsx`)
- Behavior: displays `you@gmail.com` (truncated to 24 chars with `…` if longer), or `NOT CONNECTED` link
- Placement: top-right of fixed nav bar, replacing or appending after the existing `Analyze a Ticker →` CTA button
- State: connected (amber email text), not-connected (muted link)
- Data: from extended `GET /api/setup/status` → `userEmail` field

#### `<ReportHistory />`
- Location: `src/components/ReportHistory.tsx`
- Placement: `src/app/page.tsx` — below `<TickerSearch />` (or `<SetupWizard />`), replacing the "How it works" section
- Layout: full-width section with `.panel` container and `#1a2d42` border
- States: loading skeleton (3 greyed rows), empty state message, populated list

#### History row structure
Each row is a single horizontal line using `grid` layout with fixed column widths:
```
SYMBOL    | COMPANY              | DATE        | SENTIMENT | CONFIDENCE | OPEN | REGEN
80px      | flex-1               | 110px       | 90px      | 80px       | 48px | 56px
```
- SYMBOL: amber `#f59e0b`, 11px bold, tracking-[0.1em]
- COMPANY: `#4a6a8a`, 11px, truncated with `overflow-hidden text-ellipsis`
- DATE: `#3d5e7a`, 10px, tabular-nums — format: `Mar 18 2026`
- SENTIMENT chip: `BULLISH` (#10b981 on #064e3b bg), `BEARISH` (#ef4444 on #2d0a0a bg), `NEUTRAL` (#3d5e7a on #0a1520 bg) — 8px font, 2px 6px padding, no border-radius (sharp terminal style)
- CONFIDENCE chip: `HIGH` / `MEDIUM` / `LOW` in `#3d5e7a` muted — 8px, no background
- `[OPEN]`: amber text, 9px, no background, hover: amber glow text-shadow
- `[REGEN]`: muted `#3a5a78` text, 9px, no background, hover: `#6b8fa8`

## Interaction Contracts

### Nav Identity
- On page load: fetch `GET /api/setup/status` (already called for setup check) → read `userEmail` from response
- Connected: display truncated email in amber — no hover action
- Not connected: display `NOT CONNECTED` as interactive link — click scrolls to `#setup` anchor (SetupWizard)
- Truncation rule: if email length > 24 chars, show first 21 chars + `…`

### History List
- On page load (after setup check resolves): fetch `GET /api/history` → populate list
- Loading: render 3 placeholder rows with `opacity-30` static content (no spinner — terminal aesthetic)
- Empty state: single row spanning full width — `No reports yet. Analyze a ticker to get started.` — `#3a5070` color, 10px, tracking-[0.2em]
- `[OPEN]` click: navigate to `/research/[ticker]?report=[filename]` — loads saved AnalysisResult without re-running pipeline
- `[REGENERATE]` click: navigate to `/research/[ticker]` — starts full pipeline for that ticker (chart confirmation included)
- No confirmation dialog for REGENERATE — old reports are preserved (non-destructive operation)
- Hover: row background transitions to `#0d1420` over 120ms (`transition: background 0.12s` matching `.market-grid-row` pattern)

### Report Persistence (API side — visible to UI)
- After analysis completes: report appears at top of history list on next page load (or after manual refresh)
- No real-time push — user sees new report after returning to home page

| Element | Copy |
|---------|------|
| Primary CTA — Open report | `[OPEN]` |
| Primary CTA — Regenerate | `[REGENERATE]` |
| Section header | `RESEARCH HISTORY` |
| Column headers | `SYMBOL` / `COMPANY` / `DATE` / `SENTIMENT` / `CONFIDENCE` |
| Empty state (single-line) | `No reports yet. Analyze a ticker to get started.` |
| Loading state | `LOADING HISTORY...` (10px, tracking-[0.3em], muted `#2a4560`) |
| Not connected label | `NOT CONNECTED` |
| Auth error in nav | `NOT CONNECTED` (same copy — no verbose message) |
| History fetch error | `HISTORY UNAVAILABLE` — inline in the section where list would appear |
| Permissions error (dir creation fails) | Surface in history section: `Storage unavailable. Check ~/.equinfo/ permissions.` |

**Tone:** All copy follows terminal command/status conventions — uppercase labels, no punctuation on standalone status strings, lowercase only for inline body descriptions.

## Sentinel States

All states that must be handled with explicit UI:

| State | Component | Treatment |
|-------|-----------|-----------|
| Setup not complete | `<NavIdentity />` | Show `NOT CONNECTED` — no email until auth succeeds |
| History loading | `<ReportHistory />` | 3 greyed skeleton rows (opacity-30, static content, no animation) |
| History empty | `<ReportHistory />` | Single empty-state row spanning full width |
| History fetch error | `<ReportHistory />` | `HISTORY UNAVAILABLE` inline message |
| Long email (>24 chars) | `<NavIdentity />` | Truncate at 21 chars + `…` |
| No reports for regeneration target | n/a | Regenerate navigates to `/research/[ticker]` — pipeline handles the rest |
