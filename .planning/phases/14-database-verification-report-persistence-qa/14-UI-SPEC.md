## Component Contract: ReportHistory.tsx

### Bug Fix: OPEN button navigation

The existing `toFilename()` function produces `TICKER-YYYY-MM-DDTHH-MM-SSZ.json`. In web mode
this string is passed as `?report=` to `/research/[ticker]?report=`. The history API route
treats this param as a UUID. Because the constructed filename is not a UUID, every OPEN click
returns 404.

**Required change — navigation key logic:**

```
const navKey = report.id ?? toFilename(report);
```

- Web mode: `report.id` is populated (UUID from Neon) → used as the `?report=` value
- Local mode: `report.id` is undefined → falls back to `toFilename(report)` (unchanged behavior)

The `key` prop on each row must also use `navKey` (or `report.id ?? filename`) so React
correctly reconciles rows.

**Required change — `data-testid` attributes (must not be removed):**

| Element | data-testid |
|---------|-------------|
| Each history row `<div>` | `history-row` |
| OPEN `<button>` | `history-open-btn` |
| REGEN `<button>` | `history-regen-btn` |

These testids are referenced by the Playwright e2e spec in this phase — they must survive the
bug fix unchanged.

### Row layout (unchanged)

7-column CSS grid: `grid-cols-[80px_1fr_110px_90px_80px_48px_56px]`

| Col | Content | Color | Size/Weight |
|-----|---------|-------|-------------|
| 1 | Ticker symbol | `#f59e0b` amber | 11px bold |
| 2 | Company name (ellipsis overflow) | `#4a6a8a` | 11px regular |
| 3 | Date (tabular-nums) | `#3d5e7a` | 10px regular |
| 4 | Sentiment badge | per-sentiment color | 8px bold uppercase |
| 5 | Confidence level | `#3d5e7a` | 8px uppercase |
| 6 | [OPEN] button | `#f59e0b` default, `#fbbf24` hover | 9px |
| 7 | [REGEN] button | `#3a5a78` default, `#6b8fa8` hover | 9px |

Row height: 40px (inline style — do not convert to Tailwind).
Row hover background: `#0d1420` (inline style — do not convert to Tailwind).

### States

| State | Visual |

| loading | 3 skeleton rows at 30% opacity, each with a single `LOADING HISTORY...` label or horizontal `h-px` dividers |
| error | Single row showing `HISTORY UNAVAILABLE` in `#2a4560` |

## Component Contract: ResearchReport.tsx

### Backward-compatibility guards for Phase 12/13 fields

All Phase 12/13 additions to `AnalysisResult` are typed as optional (`?`). Any destructuring
or JSX reference to these fields must be guarded so a pre-Phase-12 report (missing these
fields entirely) does not throw or crash.

**Required pattern — all of the following fields must use optional chaining or null-coalescing
before any property access or rendering:**

| Field | Guard pattern |
|-------|---------------|
| `sentiment_intelligence` | `analysisResult.sentiment_intelligence?.stocktwits_bull_pct` |
| `sentiment_intelligence.put_call_ratio` | `?.put_call_ratio` |
| `sentiment_intelligence.put_call_interpretation` | `?.put_call_interpretation` |
| `future_projection` | `{future_projection && <section>...</section>}` |
| `price_target` | `{price_target && <span>...</span>}` |
| `community_highlights` | `{community_highlights && community_highlights.length > 0 && ...}` |
| `community_analysis` | `{community_analysis && ...}` |
| `community_sources_scraped` | `{community_sources_scraped != null && ...}` |

**Rule:** If an entire section is gated on a Phase 12/13 field, the section must not render at
all when the field is absent (undefined or null). Do not render headings, labels, or placeholder
text for absent sections. Silently omit.

**Rule:** Any field rendered inside a section that could be null must use `?? '—'` or
`?? null` rather than rendering undefined directly into JSX.

**Existing conditional rendering in ResearchReport.tsx that must be preserved unchanged:**

- `getSentimentBadgeClass()` — already guards undefined via `getSentimentLabel()`
- `getPutCallColor()` — already handles null/undefined
- `confidenceToPercent()` — receives typed `'Low' | 'Medium' | 'High'`; no change needed
- Stats grid `?.` chains on `market_snapshot` fields — already guarded; do not remove

## Interaction Contract

### OPEN button — fixed behavior

Before fix: `router.push('/research/${ticker}?report=${encodeURIComponent(toFilename(report))}')` — produces 404 in web mode.

After fix: `router.push('/research/${report.ticker}?report=${encodeURIComponent(navKey)}')` where `navKey = report.id ?? toFilename(report)`.

No visual change to the button — only the navigation target changes.

### REGEN button — unchanged

`router.push('/research/${report.ticker}')` — no change.

### Row hover — unchanged

`onMouseEnter`: set `background = '#0d1420'`
`onMouseLeave`: reset `background = ''`

Both remain as inline style mutations (not class toggling). Do not convert to Tailwind
`hover:` utilities — the transition (`transition: background 0.12s`) must be preserved via
inline style.

### Accessibility

- Both [OPEN] and [REGEN] are `<button>` elements — keyboard focusable by default.
- No additional aria labels required for this bug fix (existing state is carried forward).
- `data-testid` attributes must be preserved on all interactive elements (see table above).
