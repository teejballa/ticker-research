<objective>
Land the engine's dual-class learning loop. Two cron routes change in lockstep:

1. **`sentiment-scan`** — additionally compute `TechnicalSnapshot` per ticker and write it on the SAME `SentimentSnapshot` row that already carries `community_data`. One snapshot row = one full sensor reading.
2. **`learn`** — for every resolved outcome at every horizon, run **dual** Beta-cell updates (`(diffusion, flow_pattern, cap_class, horizon)` + `(technical, tech_pattern, cap_class, horizon)`); train the **12-feature Bayesian logistic** on **30d outcomes only**; recompute Brier/drift across all **216 cells** (4 diffusion + 8 technical patterns × 3 cap_classes × 6 horizons; the prior 288 figure assumed 4 cap_classes — the actual `classifyCapClass()` union has 3 traded buckets plus an `unknown` fallback that is not learned). Wrap each per-outcome work block in `prisma.$transaction` so retries never double-count. Reinitialize the logistic from scratch on first post-Phase-16 cycle (Pitfall 5).

Purpose: This plan is the engine's behavioral pivot. Plan 16-01 produced the sensor; plan 16-02 produced the schema; plan 16-03 makes the engine USE both. After this plan, the engine writes dual snapshots and learns dual posteriors automatically.

Output: Crons that write + learn, integration tests proving the dual-class invariants, and `learning.ts` updated to recognize 12 feature names.
</objective>
