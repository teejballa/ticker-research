<objective>
Reshape the engine's persistence + outcome layer for dual-class learning at six horizons. (1) Migrate `LearnedPattern` from `(flow_pattern × cap_class)` keyed cells to `(signal_class × pattern_key × cap_class × horizon_days)` keyed cells, atomically backfilling existing rows as `signal_class='diffusion'`, `horizon_days=7`. (2) Add nullable JSON columns `SentimentSnapshot.technical_data` and `Report.technical_at_report`. (3) Push the schema to live Neon so types regenerate against reality. (4) Extend `price-followup/route.ts` from horizons `[3, 7, 14]` to `[3, 7, 14, 30, 60, 90]` and widen the query window from 15d to 95d. (5) Land integration tests proving the migration and the new horizons.

Purpose: This is the engine-wide schema reshape. Without this, plan 16-03's dual-class learn loop has no cells to write to and no 30/60/90d outcomes to learn from. Plan 16-04's `engine-context.ts` extension queries the new columns directly. Plan 16-05's backfill writes through this schema.

Output: Live Neon schema in the post-Phase-16 shape, the cron extended to write 6 horizons, and live-DB tests that prove both.
</objective>

```sql
-- 1. Add new columns to learned_patterns (with DEFAULTs so existing rows backfill atomically)
ALTER TABLE "learned_patterns" ADD COLUMN "signal_class" TEXT NOT NULL DEFAULT 'diffusion';
ALTER TABLE "learned_patterns" ADD COLUMN "horizon_days" INTEGER NOT NULL DEFAULT 7;
ALTER TABLE "learned_patterns" ADD COLUMN "pattern_key" TEXT;

-- 2. Backfill pattern_key from flow_pattern (BEFORE the NOT NULL constraint)
UPDATE "learned_patterns" SET "pattern_key" = "flow_pattern";

-- 3. Make pattern_key NOT NULL and drop flow_pattern + old unique constraint
ALTER TABLE "learned_patterns" ALTER COLUMN "pattern_key" SET NOT NULL;
ALTER TABLE "learned_patterns" DROP CONSTRAINT IF EXISTS "learned_patterns_flow_pattern_cap_class_key";

-- 4. Add the new unique index
CREATE UNIQUE INDEX "learned_patterns_signal_class_pattern_key_cap_class_horizon_days_key"
  ON "learned_patterns"("signal_class", "pattern_key", "cap_class", "horizon_days");

-- 5. Drop the DEFAULTs (they were just to enable atomic backfill)
ALTER TABLE "learned_patterns" ALTER COLUMN "signal_class" DROP DEFAULT;
ALTER TABLE "learned_patterns" ALTER COLUMN "horizon_days" DROP DEFAULT;

-- 6. Add JSON columns to other tables
ALTER TABLE "sentiment_snapshots" ADD COLUMN "technical_data" JSONB;
ALTER TABLE "reports" ADD COLUMN "technical_at_report" JSONB;
```
