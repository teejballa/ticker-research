technicalindicators v3 API gotchas (RESEARCH.md §3.1 lines 470-488):
- All four (RSI/MACD/SMA/ATR) expose static `.calculate()`.
- Output arrays are TRUNCATED, not padded — RSI(14) on 250 closes returns 236 values. ALWAYS read `arr[arr.length - 1]` for "most recent".
- MACD output `Array<{ MACD: number, signal?: number, histogram?: number }>` — `signal` and `histogram` undefined for first `signalPeriod-1` bars after slow EMA warms up. Guard with `?? null`.
- ATR uses Wilder smoothing (matches industry convention).
- yahoo-finance2 returns `number | null` for OHLCV — drop bars where ANY of high/low/close is null BEFORE passing to ATR (library does NOT handle nulls).
- Volume halt (volume === 0) — filter from 20-day avg AND set `volume_ratio = null` when latest bar is 0 (RESEARCH.md Pitfall 7 lines 939-944).

Classifier thresholds (RESEARCH.md §3.3 lines 490-500):
- `golden_cross` — sign(sma50_today - sma200_today) > 0 AND sign(sma50_yesterday - sma200_yesterday) <= 0
- `death_cross` — sign(sma50_today - sma200_today) < 0 AND sign(sma50_yesterday - sma200_yesterday) >= 0
- `overbought_uptrend` — RSI > 70 AND price > SMA200 (and not a cross)
- `oversold_downtrend` — RSI < 30 AND price < SMA200 (and not a cross)
- `breakout_uptrend` — MACD histogram > 0 AND volume_ratio > 1.5 AND price > SMA50 (above50 && above200)
- `pullback_in_uptrend` — price > SMA200 AND price > SMA50 AND RSI between 40 and 55
- `consolidation` — fallback inside an uptrend OR mixed stack
- `breakdown` — price < SMA50 AND price < SMA200 AND RSI >= 30
